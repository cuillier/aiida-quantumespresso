# Writing and building documentation for aiida-quantumespresso

When writing or editing `.md` or `.rst` files under `docs/source/`:

- Write **one sentence per line** (no manual line wrapping): makes diffs easy to review.
- File/directory names: alphanumeric, lowercase, underscores as separators.
- Headers in **sentence case** (e.g., "Entry points", not "Entry Points").
- Documentation follows the [Divio documentation system](https://www.divio.com/blog/documentation/): tutorials (learning-oriented), how-to guides (goal-oriented), topics (understanding-oriented), reference (information-oriented).

## Doc tree

```
docs/source/
├── get_started/    # installation and first steps
├── howto/          # goal-oriented guides (run a workflow, use protocols, …)
├── reference/      # API reference (auto-generated) and exit codes
├── topics/         # conceptual explanations (protocols, parsers, …)
└── tutorials/      # step-by-step learning examples
```

The `reference/api/` subtree is auto-generated by `sphinx-autoapi` from the source code and is **not committed**.
It is wiped by `make clean` and rebuilt on each doc build.

## Building the docs

```bash
uv run make -C docs html              # standard build
uv run make -C docs customdefault     # strict build: -nW --keep-going (treats warnings as errors)
uv run make -C docs clean             # wipe build artefacts including auto-generated API docs
```

The built HTML lands in `docs/build/html/`; open `docs/build/html/index.html` to preview locally.
`make -C docs view` opens it automatically on macOS.

The docs optional-dependency group must be installed:

```bash
uv sync --extra docs
```
